NeoPG Documentation

NeoPG implements the OpenPGP standard.

NeoPG is written in C++11 and released under the permissive Simplified BSD license (the same license as Botan, the cryptography library).

Some (many) parts of NeoPG are currently under a more restrictive license, because they are derived from other projects (in particular GnuPG). Please refer to the copyright notice at the top of every file.

The dependencies are also released under their respective various licenses.

Dependencies

Aside from a working C++ toolchain you'll need the following libraries.

  1. SQLite >= 3.0
  1. Botan >= 2.0
  1. CMake >= 3.2
  1. Boost >= 1.64.0
  1. openldap
  1. zlib
  1. bzip2
  1. gcovr (debug builds only)
  1. clang-format (debug builds only)
  1. cppcheck (debug builds only)
  1. doxygen (debug builds only)

Install

With all dependencies installed NeoPG can be build with CMake.

1 $ git submodule update --init
2 $ mkdir build
3 $ cd build
4 $ cmake -DCMAKE_BUILD_TYPE=Release ..
5  # or cmake -DCMAKE_BUILD_TYPE=Release -C ../src/clang.txt ..
6 $ make
7 $ make test # or ./gpg-error-test

Optional

You need to have gcovr, clang-format, doxygen and cppcheck installed.

1 $ make pretty
2 $ make lint
1 $ cmake -DCMAKE_BUILD_TYPE=Debug ..
2 $ make; make coverage # FIXME: Add proper dependencies to target.
3 $ make; make coverage-data

Hacking

Fedora

To get started on a Fedora 26, do the following

1 # Fedora 26 only comes with Botan 1.10. Here we enable a copr repo for Botan 2
2 $ sudo dnf copr enable bkircher/botan2
3 
4 # Install dev dependencies
5 $ sudo dnf install -y \
6  boost-devel \
7  botan2-devel \
8  cmake \
9  gcc-c++ \
10  gcovr \
11  git \
12  lcov \
13  libusb-devel \
14  python \
15  sqlite-devel \
16 
17 # Clone repo and build
18 $ git clone --recursive git@github.com:zaolin/neopg.git
19 $ cd neopg/build
20 $ cmake ..
21 $ make

Have fun!
